# TaxDecod Design Constitution

## 1. Product identity

TaxDecod is not a basic tax calculator.

TaxDecod is a UK salary intelligence platform that helps people understand what salary really means after tax, National Insurance, student loans, pensions, salary sacrifice, payslip deductions, and real-life monthly pressure.

The core product promise is:

> Know exactly what your salary really pays.

The product must feel calm, intelligent, useful, premium, and trustworthy.

It should guide the user through salary decisions instead of overwhelming them with numbers.

---

## 2. Design feeling

TaxDecod should feel:

- calm
- soft
- advanced
- premium
- precise
- editorial
- human
- trustworthy
- modern without looking trendy
- data-rich without feeling crowded

TaxDecod should not feel:

- generic SaaS
- cyber/neon
- navy-blue fintech template
- cheap calculator site
- cluttered affiliate blog
- random AI-generated landing page
- over-animated portfolio site
- government-site clone
- dark-mode-only crypto dashboard

---

## 3. Visual direction

The design should use a warm, calm, progressive palette.

Preferred visual language:

- warm ivory backgrounds
- soft graphite text
- muted coral/clay accent
- soft mint or sage positive states
- gentle lavender-grey secondary accents
- warm gold only as a restrained trust highlight
- soft dark mode using charcoal, not pure black or navy
- layered surfaces with subtle depth
- high-quality spacing and typography
- comfortable mobile layouts
- strong numeric hierarchy

Avoid:

- neon blue
- harsh gradients
- heavy glassmorphism
- noisy animated blobs
- excessive shadows
- too many bordered cards
- tiny text
- long uncontrolled link lists
- generic “modern startup” sections

---

## 4. Motion principles

Motion should make the calculator feel alive and guided.

Use motion for:

- result transitions
- salary breakdown changes
- monthly/yearly toggles
- card hover lift
- mobile menu reveal
- trust panel reveal
- tool step transitions
- subtle background response
- page section entrance only when meaningful

Avoid motion for:

- constant moving backgrounds
- random floating objects
- gimmicky hero effects
- animation that delays calculator use
- animation that makes numbers harder to read
- animation that harms mobile performance

Motion must feel like the interface is responding to the user, not showing off.

---

## 5. Typography rules

The typography must feel editorial and precise.

Rules:

- headings should be confident, calm, and readable
- numbers should have strong hierarchy
- body text should be easy for beginners
- labels should be short and useful
- avoid vague AI marketing language
- avoid overused words like “unlock”, “empower”, “seamless”, “revolutionise”
- use plain English for tax concepts
- use professional wording for trust/legal sections

Page titles should explain the user benefit quickly.

---

## 6. UX principles

Every important page should answer:

1. What can I do here?
2. What number should I enter?
3. What result matters most?
4. What does the result mean?
5. What should I check next?
6. What assumptions are being used?
7. Why should I trust this?

The user should never feel lost.

Each tool page should include:

- clear hero
- calculator/input section
- result explanation
- deduction or comparison breakdown
- assumptions/trust strip
- useful next-step links
- FAQ or explanatory section where appropriate
- disclaimer where appropriate

---

## 7. Trust principles

TaxDecod is finance-adjacent and tax-related, so it must be careful.

Every serious page should show:

- relevant UK tax-year context
- methodology/assumptions link
- plain disclaimer
- no fake guarantees
- no misleading “official” claims
- no pretending to replace HMRC, payroll, accountant, or financial adviser
- no saving personal salary data unless clearly explained and consented

Trust must be built through clarity, not through fake badges.

---

## 8. SEO quality principles

No page should exist just because AI can generate it.

A page should be indexable only if it has:

- unique intent
- useful calculation or explanation
- clear page title
- specific salary/tax context
- internal links that make sense
- no thin duplicated paragraphs
- no fake authority claims
- enough value for a real UK user

Programmatic pages must be controlled by approved config.

Sitemap inclusion is a quality signal. A page being live does not automatically mean it belongs in the sitemap.

---

## 9. Component principles

TaxDecod should be built from reusable premium components:

- App shell
- Header
- Footer
- Tool shell
- Calculator card
- Result panel
- Deduction breakdown
- Trust strip
- CTA cluster
- Editorial block
- FAQ block
- Cross-link rail
- Comparison card
- Salary insight card
- Methodology panel

Do not redesign the same pattern differently on every page.

One strong component should improve many pages.

---

## 10. Dark and light mode

Both modes must feel intentional.

Light mode:

- warm, calm, clean
- not sterile white
- soft surfaces
- strong readability

Dark mode:

- charcoal, graphite, warm black
- not navy
- not neon
- not cyberpunk
- comfortable for long reading
- clear contrast for numbers and forms

Dark mode cannot be treated as an afterthought.

---

## 11. Mobile rules

Mobile is a priority.

Rules:

- no cramped calculator layouts
- no tiny result cards
- no horizontal scrolling
- no huge hero that hides the actual tool
- sticky or easy access to main action where appropriate
- navigation must be simple and fast
- forms must be comfortable to tap
- results must be readable without zooming

Most real users will judge the product on mobile first.

---

## 12. Redesign workflow rules

Every redesign patch must follow this order:

1. Plan the change
2. Identify affected files
3. Replace full files only when needed
4. Run build
5. Test key pages locally
6. Commit with clear message
7. Do not merge to main until recovery and redesign are approved

Do not allow AI tools to rewrite the whole site without a scoped file plan.

Do not change routing casually.

Do not delete indexed pages casually.

Do not expose backend keys or user data.

---

## 13. TaxDecod design north star

TaxDecod should feel like a calm financial intelligence product that translates confusing salary and tax numbers into clear decisions.

The best version of TaxDecod is not louder.

It is clearer, softer, faster, more trustworthy, and more useful.
